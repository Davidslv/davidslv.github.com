Dashboard

Live Demo

About

AI Dashboard is a single-page web application that helps users discover, search, and organize AI tools and services. It displays services in a smartphone-style grid layout with hover descriptions, category filtering, and search to find tools across categories like Video Generation, Automation, Research, Development, and more.

Motivation

It was built to keep services I use or want to use in one place, avoiding browser bookmark bloat and forgotten links. Instead of scattered bookmarks, it provides a single hub for discovering and accessing AI tools with an intuitive, app-like experience.

Tech Stack

Built with React 18, Vite for fast development and builds, CSS Modules for component-scoped styling, and vanilla JavaScript. Fully client-side with no backend—all data is statically defined, making it fast, simple, and easy to deploy.

Microblog

Live Demo

About

Microblog is a high-performance microblogging platform where users post short messages (200 characters), reply to create conversations, follow other users, and view personalized feeds (timeline, following, or all posts). It uses modern techniques to handle data-intensive operations such as fan-out on write and horizontal scaling.

Motivation

Built as a production-ready educational case study demonstrating scalable Rails architecture, performance optimization techniques, and modern web development practices. Shows real-world engineering from database design to horizontal scaling, serving as a practical learning resource for developers.

Tech Stack

Built with Ruby on Rails 8.1.1 backend (PostgreSQL, Puma, Solid Queue, JWT auth) and React 18+ frontend (Vite, Tailwind CSS, Axios). Follows a three-layer architecture (Presentation/Application/Data) enabling independent scaling. Uses RSpec, Vitest, and Playwright for testing.

Pokédex

Live Demo

About

A browser-based recreation of the classic red Pokédex from Pokémon Gen 1, featuring 151 Pokémons. This interactive web app provides a nostalgic, kid-friendly encyclopedia with authentic design and voice narration. Users can browse Pokémon, view detailed stats, types, evolution chains, and hear auto-narrated descriptions.

Motivation

Built as a labor of love for my nephew, this project combines family bonding with web development passion. It taught me user-centered design by focusing on what excites a 7-year-old: vibrant visuals, simple controls, and interactive audio.

Tech Stack

Vanilla HTML5, CSS3, and JavaScript for a lightweight, framework-free experience.

Vanilla Roguelike

GitHub

About

Vanilla is a terminal-based roguelike game engine written in Ruby featuring procedurally generated mazes, player navigation, monster AI, and inventory systems. It uses Entity-Component-System architecture with event-driven design for game state tracking and comprehensive debugging capabilities.

Motivation

After building Vanilla and learning valuable lessons about game architecture, I documented my experience in "Building Your Own Roguelike". The book distills practical insights from this real-world implementation, making complex patterns like ECS architecture, command patterns, and event-driven systems accessible to developers learning game development.

Tech Stack

Built with Ruby 3.4.1, using RSpec for testing, I18n for localization, and SimpleCov for coverage. Implements ECS pattern with modular systems (movement, rendering, AI), event-driven logging with visualization tools, and multiple maze generation algorithms (Binary Tree, Aldous-Broder, Recursive Backtracker, Recursive Division).

About

Manuscript Forge is a work-in-progress platform designed to help writers manage, organize, and refine their manuscripts. It provides tools for version control, collaboration, and structured writing workflows.

Motivation

Created after writing "Vanilla Roguelike" in Markdown and needing proper chapter and diagram formatting for PDF export and Amazon KDP. Existing tools couldn't handle structured conversion, so I built custom tooling to convert Markdown to DOCX for formatting control, enabling PDF export and KDP ebook creation.

Tech Stack

Built with Ruby on Rails 8.1 (API mode), PostgreSQL 16, Solid Queue for async processing (Postgres-backed, no Redis), Shrine + AWS S3 for file storage, pandoc-ruby for document conversion, React + Vite frontend, Tailwind CSS, and RSpec for testing. Architecture follows service-oriented design with background jobs for document processing.

Cashew

Live Demo

About

Cashew is a single-page web application that helps UK users compare mortgage overpayment strategies against investment alternatives like Cash ISAs and Stocks & Shares ISAs. It calculates interest savings, term reductions, investment growth projections, and factors in UK tax implications to show which strategy is financially better.

Motivation

I created Cashew to address the need for a simple, privacy-focused financial decision tool. Many existing solutions are either too complex or compromise user privacy. Cashew aims to provide powerful financial insights while keeping user data secure and private—no data storage, fully GDPR-compliant, with all calculations happening in real-time.

Tech Stack

Built with Ruby on Rails 8.1.1, using Hotwire (Turbo + Stimulus) for SPA behavior, Importmap for JavaScript modules, RSpec for testing, and Chart.js for visualizations. The architecture follows SOLID principles with service-oriented design.
